ขออนุญาตส่ง Assignment โปรเจกต์ SecureNote Application ครับ โดยระบบได้รับการพัฒนาตามสถาปัตยกรรมที่กำหนด พร้อมทั้งบูรณาการส่วน Cloud Deployment และ Hybrid Data Persistence เป็นที่เรียบร้อยแล้วครับ

1. ช่องทางการเข้าถึงโค้ดต้นฉบับ
สามารถตรวจสอบโครงสร้างโค้ด เอกสาร README.md ได้ที่ GitHub Repository:
-> https://github.com/Knnthiti/SecureNote

2. ช่องทางการทดสอบระบบในเครื่อง
ระบบพัฒนาและทดสอบบนสภาวะแวดล้อมภายในเครื่อง (Frontend บน Next.js, Backend บน Express)
-> Frontend: http://localhost:3000
-> Backend API: http://localhost:3001
-> System Token สำหรับทดสอบระบบ (Dynamic Authentication):
    • ทดสอบ Public Mode (Local FS): กรอก Token admin123
    • ทดสอบ Instructor Mode (PocketHost API): กรอก Token 20260301eink แล้วสลับ Radio เป็น "PocketHost API"
(หมายเหตุ: ระบบบังคับใช้การตรวจสอบสิทธิ์เสมอสำหรับการสร้าง แก้ไข และลบโน็ต แต่ได้แยก Token ให้สอดคล้องกับฐานข้อมูลปลายทางในแต่ละโหมดอย่างอิสระ)

3. สถาปัตยกรรมข้อมูลและวิธีการตรวจประเมิน
ระบบถูกออกแบบให้รองรับการจัดเก็บข้อมูล 2 รูปแบบ ซึ่งควบคุมการสลับโหมดผ่าน Radio Button บนหน้า UI:
-> Public Mode (Local FS): ประมวลผลและจัดเก็บข้อมูลลงไฟล์ notes.json ผ่าน Node.js fs module
-> Instructor Mode (PocketHost API): ระบบ Backend จะทำหน้าที่เป็น Multiplexer (Proxy) เพื่อจัดการ Routing ไปยังฐานข้อมูล PocketHost ตามที่ระบุในโจทย์
    • วิธีการตรวจประเมิน Instructor Mode: กดปุ่มสลับเป็น "PocketHost API" บนหน้าเว็บ และกรอก System Token 20260301eink เพื่อทดสอบฟังก์ชัน Create, Update และ Delete ได้ทันที (ระบบ Backend ได้รับการตั้งค่าให้จัดการ Format ของ Authorization ด้วยการเติมคำว่า Bearer และแทรกค่า user_id: 2 ลงใน Payload ให้อัตโนมัติ เพื่อให้สอดคล้องกับ Schema ฐานข้อมูลของอาจารย์)

4. ข้อจำกัดทางสถาปัตยกรรมคลาวด์
-> Cold Start Latency: เนื่องจาก Backend ถูกติดตั้งบนแพลตฟอร์ม Render (Free Tier) คอนเทนเนอร์จะถูกระงับการทำงาน (Spin down) หากไม่มีการใช้งาน 15 นาที หากเกิดข้อผิดพลาด "Failed to load notes" ในการเรียกใช้งานครั้งแรก กรุณารอ 30-60 วินาทีเพื่อให้ระบบตื่นตัว แล้วรีเฟรชหน้าเว็บอีกครั้ง
-> Ephemeral Storage: สำหรับการทดสอบ Public Mode บนสภาวะแวดล้อม Live Production ข้อมูลที่ถูกเขียนลงไฟล์ notes.json จะสูญหายเมื่อ Render เข้าสู่สภาวะ Spin down หรือรีสตาร์ท เนื่องจากข้อจำกัดด้านพื้นที่จัดเก็บชั่วคราวของระบบ Cloud การจัดเก็บข้อมูลแบบถาวรบน Live Production จะทำงานสมบูรณ์เฉพาะใน Instructor Mode เท่านั้น

5. ขั้นตอนการเรียกใช้งานบนเครื่องส่วนตัว
หากต้องการคอมไพล์และทดสอบระบบในสภาวะแวดล้อมจำลอง กรุณาดำเนินการตามขั้นตอนต่อไปนี้:

ส่วนที่ 1: การตั้งค่า Backend
• เปิด Terminal และนำทางไปยังไดเรกทอรี: cd backend
• ติดตั้ง Dependency: npm install
• สร้างไฟล์ .env และกำหนดค่าตัวแปร:
    -- PORT=3001
    -- SECRET_TOKEN=admin123
    -- POCKET_HOST_URL=https://app-tracking.pockethost.io/api/collections/notes/records
    -- POCKET_HOST_TOKEN=20260301eink
    -- POCKET_HOST_USER_ID=2
• เริ่มการทำงานของเซิร์ฟเวอร์: npm start (ระบบจะทำงานบน http://localhost:3001)

ส่วนที่ 2: การตั้งค่า Frontend
• เปิด Terminal หน้าต่างใหม่ และนำทางไปยังไดเรกทอรี: cd frontend
• ติดตั้ง Dependency: npm install
• สร้างไฟล์ .env.local และกำหนดค่าตัวแปร:
    -- NEXT_PUBLIC_SECRET_TOKEN=admin123
    -- NEXT_PUBLIC_POCKET_TOKEN=20260301eink
• เริ่มการทำงานของไคลเอนต์: npm run dev (ระบบจะรันบน http://localhost:3000)
